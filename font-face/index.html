Id de fontes SVG e o Font Squirrel
  • Referência

Há um problema quando tentamos usar diversas fontes SVG do Font Squirrel, pois estas simplesmente não funcionam no navegador. O trecho de CSS fornecido para uso é similar a este: 

@font-face {
	font-family: 'GoodDogRegular';
	src: url('GoodDog-webfont.svg#GoodDogRegular') format('svg');
	font-weight: normal;
	font-style: normal;
}

O problema está na referência pro id da fonte, no caso #GoodDogRegular. Abrindo o arquivo encontramos a entrada <font id="algumaCoisa" … que contém o verdadeiro id da fonte:

<?xml version="1.0" standalone="no"?>
<!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN" "http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd" >
<svg xmlns="http://www.w3.org/2000/svg">
<metadata>
This is a custom SVG webfont generated by Font Squirrel.
Copyright   : Copyright c 1996 by Ethan Dunham  Fonthead Design All rights reserved
Designer    : Ethan Dunham
Foundry     : Ethan Dunham  Fonthead Design
Foundry URL : httpwwwfontheadcom
</metadata>
<defs>
<font id="webfonthc2mR75o" horiz-adv-x="735" >

Substituindo o id a chamada CSS correta fica

@font-face {
	font-family: 'GoodDogRegular';
	src: url('GoodDog-webfont.svg#webfonthc2mR75o') format('svg');
	font-weight: normal;
	font-style: normal;
}
